# WGSL4K-EVO-004 - Consume wgsl4k reflection in Kanvas reports

## PM Note

Ce ticket branche les rapports Kanvas sur wgsl4k sans promouvoir de route.

## Problem

Blocked Kanvas tickets need fresh, machine-readable validation/reflection
reports generated from a reviewed wgsl4k SHA. Manual claims or syntax-only
checks are not enough.

## Scope

- Consume the reviewed wgsl4k SHA in Kanvas validation tooling.
- Emit or update text WGSL reports such as `text-wgsl-reflection.json` and
  `text-wgsl-validation-report.json`.
- Emit or update registered runtime-effect WGSL reflection reports.
- Compare wgsl4k reflection with Kanvas binding, packing, resource, and
  descriptor plans.
- Keep route promotion for follow-up owning tickets.

## Non-Goals

- Do not promote A8/SDF/color text routes in this ticket.
- Do not promote registered runtime effects in this ticket.
- Do not treat wgsl4k success as CPU/GPU/reference evidence.

## Acceptance Criteria

- [ ] Kanvas records the wgsl4k SHA used for reports.
- [ ] Text WGSL reflection reports include module hash, entry points, bindings,
  layouts, comparison result, and diagnostics.
- [ ] Runtime-effect WGSL reflection reports include descriptor id/version,
  module hash, entry points, bindings, layouts, comparison result, and
  diagnostics.
- [ ] Negative report fixtures cover parser failure, binding mismatch, layout
  mismatch, and unregistered module refusal.
- [ ] No product route support claim changes.

## Required Evidence

- Fresh report fixtures generated by Kanvas.
- Targeted tests for reflection report generation and comparison failures.
- `rtk git diff --check`.

## Fallback / Refusal Behavior

Parser, reflection, or comparison failures refuse the report consumer path with
stable diagnostics. They do not fall back to unvalidated shader code.

## Validation

```bash
rtk ./gradlew --no-daemon :font:gpu-api:test --tests '*TextWgsl*'
rtk ./gradlew --no-daemon :gpu-renderer:test --tests '*Wgsl*'
rtk git diff --check
```

Exact test targets may be refined by the implementation plan once the
consuming modules are selected.

## Status Notes

  `https://github.com/ygdrasil-io/wgsl4k/pull/9` or an explicit user-approved
  commit before Kanvas can generate/compare report fixtures. No Kanvas reports
  were generated in this ticket wave.
