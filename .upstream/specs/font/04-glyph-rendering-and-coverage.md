# Glyph Rendering And Coverage

Status: Draft
Target: `.upstream/target/skia-like-realtime-renderer-target.md`

## Purpose

Define how glyphs become renderable artifacts in Kanvas. This file owns glyph
outline rendering, glyph mask/SDF/LCD policy, atlas ownership, CPU/WebGPU handoff,
and font-specific diagnostics.

It does not own generic path coverage, paint blending, or non-text masks except
where a glyph representation hands off to those systems.

## Current Outline Path Route

The currently supported simple route is:

```text
SkFont / SkTypeface
  -> glyph path
  -> positioned text path
  -> SkCanvas.drawPath
  -> CPU raster or WebGPU path pipeline
```

This route is valid for simple outline glyphs and for color glyph layers that
are represented as ordered outline paths.

It is not a GPU glyph atlas route and does not claim Skia Ganesh/Graphite text
op parity.

## Current WebGPU Text Evidence

Current WebGPU text evidence is smoke-level and route-specific:

| Test | Claim |
|---|---|
| `TextSmokeWebGpuTest` | Simple outline text reaches WebGPU path rendering and produces visible pixels. |
| `TextMaskFilterWebGpuTest` | `drawString` and `drawTextBlob` text paths honor blur mask filters on WebGPU. |

These tests are useful but not enough for broad font readiness. Promoted font
support scenes need reference, CPU, GPU, diff/stat, and route artifacts.

## Glyph Representation Routes

| Route | Current status | Target rule |
|---|---|---|
| `font.glyph.outline-path` | Active | Default for monochrome TrueType outline glyphs. |
| `font.glyph.colr-layer-paths` | Active for supported slices | Use only when palette/layer path evidence exists. |
| `font.glyph.bitmap-strike` | Metadata exists for selected tables | Requires internal decode, strike selection, placement, and artifacts before support. |
| `font.glyph.svg-document` | Metadata only | Refuse rendering until an internal SVG glyph renderer is accepted. |
| `font.glyph.alpha-mask` | Contract exists | Text infrastructure must produce mask refs; WebGPU currently refuses standalone alpha masks. |
| `font.glyph.sdf-mask` | Gated | Requires internal SDF generation, scale policy, CPU/GPU sampling rules, and evidence. |
| `font.glyph.lcd-mask` | Gated | Requires LCD ordering, color-fringe policy, destination transform policy, and evidence. |
| `font.glyph.unsupported` | Active as refusal shape | Stable reason explains why no glyph representation can render. |

These `font.glyph.*` values are route identifiers for serialized evidence.
They are separate from `font.*` fallback/refusal reason codes, which are
proposed until emitted by implementation evidence, and from the implemented
`coverage.*` diagnostics owned by geometry/coverage.

## Glyph Mask Ownership

The text/glyph infrastructure owns:

- glyph discovery;
- glyph rasterization;
- mask format selection;
- atlas allocation and lifetime;
- invalidation;
- glyph key construction;
- strike selection;
- cache budget and eviction policy;
- mask upload scheduling.

Geometry/Coverage owns only:

- consuming an opaque mask ref when supplied;
- representing mask bounds and format;
- reporting stable refusal or dependency diagnostics.

This preserves the geometry rule that glyph masks are not generated by the
coverage layer.

## Alpha Mask Handoff

Target handoff shape:

```text
GlyphRunDescriptor
  -> GlyphMaskRecord {
       maskRef
       bounds
       format: A8 | LCD | SDF | ARGB
       glyphKeySet
       atlasGeneration
       invalidationToken
     }
  -> CoveragePlan.AlphaMask
```

If any required field is absent, the route must emit:

```text
coverage.glyph-mask-dependency-unavailable
```

If WebGPU cannot consume the mask, it must emit:

```text
coverage.alpha-mask-unsupported
```

## SDF Text

SDF text remains gated for broad support. A valid internal SDF slice must define:

- glyph source: outline path or bitmap;
- SDF resolution and padding;
- spread radius;
- transform eligibility;
- min/max text size policy;
- CPU oracle generation;
- WebGPU sampling shader;
- LCD vs monochrome choice;
- cache key and invalidation;
- fallback reason when the run is not SDF-eligible.

Recommended reason codes:

| Code | Use |
|---|---|
| `font.sdf-text-unavailable` | SDF route was requested but generator/sampler is not available. |
| `font.sdf-transform-unsupported` | Transform is outside accepted SDF policy. |
| `font.sdf-size-unsupported` | Text size is outside SDF route bounds. |

## LCD Text

LCD text must not be claimed until Kanvas has:

- subpixel component ordering policy;
- destination color-space and gamma policy;
- transform restrictions;
- CPU/GPU reference comparisons;
- fallback to grayscale AA when LCD is unsafe;
- evidence for saveLayer and blend interactions.

Until then, LCD-related GMs can remain ratchets for current behavior without
claiming full LCD text parity.

## Glyph Atlas Policy

A future glyph atlas must define:

- glyph key;
- font identity;
- variation/palette identity;
- glyph representation;
- mask format;
- transform bucket;
- cache budget;
- eviction;
- CPU/GPU synchronization;
- stale entry invalidation;
- diagnostics for missing or invalidated entries.

No atlas can become a support dependency until it has tests for cache identity,
eviction, invalidation, and fallback behavior.

## CPU Behavior

CPU remains the font rendering oracle for Kanvas-owned behavior:

- outline glyphs render through existing path rasterization;
- masks can be produced internally only when the text/glyph layer owns them;
- glyph mask fallback must be visible and testable;
- OpenType-vs-FreeType drift must be documented, not hidden.

## WebGPU Behavior

WebGPU may support a font route only when:

- the route has a deterministic CPU oracle or upstream reference;
- a GPU artifact exists on a named adapter;
- route diagnostics state outline, mask, SDF, bitmap, or refusal;
- fallback reason is `none` for support rows;
- refusal rows carry stable reason codes.

Current WebGPU glyph-mask alpha coverage remains refused until adapter-backed
evidence proves a real mask sampling path.

## Acceptance Criteria

- Outline text route remains covered by CPU and WebGPU smoke tests.
- Glyph-mask support claims identify text-owned atlas and mask records.
- SDF/LCD/bitmap glyph routes remain gated until they have internal contracts.
- WebGPU refusal uses stable diagnostics instead of silent fallback.
- No font rendering route depends on Ganesh, Graphite, or external libraries.
