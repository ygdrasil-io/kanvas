## Interpretation

The TSV is reproducible, but several mechanical rows are too coarse for
current planning. A row can remain open while a sub-slice inside that row
has already landed.

Rows with retired or partially retired sub-slices:

| Row | Current state |
| --- | --- |
| `gradients` | Still open for LCH/OKLCH/HWB and broader color-space interpolation. HSL hue-method and HSL powerless-hue work has landed and should not be counted as pending. |
| `mesh` | Still open for GM parity. Public CPU skeleton and color-attribute support has landed; remaining blockers are shader attrs/varyings/uniforms/children, fragment output, and picture-playback coverage. |
| `vertices` | Still open for visual parity. The broad `STUB.DRAW_VERTICES` label is stale; the current actionable blocker is texCoord/local-shader sampling, especially scaled-gradient cells. |
| `colrv1` / `palette` | Still font/fixture-gated in GM planning. OpenType COLRv1 paint rendering has advanced through gradients and composites, so older "COLRv1 absent" wording is stale. |
| `SkShader.makeWithColorFilter` | Retired as an actionable API gap; implementation and GM coverage exist. Do not keep it in the active backlog. |
| image-filter multi-filter spans | Retired unless a future API-delta proves a separate overload gap; current merge/filter graph coverage supports N inputs. |
