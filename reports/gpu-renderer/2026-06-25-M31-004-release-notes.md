# KGPU-M31-004 — Release Notes and API Stability

## Release Notes — Kanvas Bridge Renderer

### Summary
Kanvas bridge renderer — supported: solid-fill rect, rrect, path; blends/gradients/strokes: to be dispatched; image/text: explicitly refused (deferred). API surface under active development.

### Supported Draw Families (via bridge)
- **drawRect** — axis-aligned rectangle fill and stroke
- **drawRRect** — rounded rectangle fill with per-corner radii
- **drawPath** — path fill with winding/even-odd/inverse fill types
- **drawImage** — explicitly refused (route kind: RefuseDiagnostic); deferred
- **drawTextBlob** — explicitly refused (route kind: RefuseDiagnostic); deferred

### Shader Types (bridge-recorder routing)
- **SolidColor** — uniform RGBA fill (dispatch ready)
- **LinearGradient** — two-point linear gradient (route kind: RefuseDiagnostic)
- **RadialGradient** — center+radius radial gradient (route kind: RefuseDiagnostic)
- **SweepGradient** — angular sweep gradient (route kind: RefuseDiagnostic)
- **Bitmap** — image-based shader (route kind: RefuseDiagnostic)

### Blend Modes
SRC_OVER only — all other blend modes are refused by the recorder. The `toKanvasBlendMode()` mapper covers 16 SkBlendMode constants (Clear through Screen), but the recorder routes all blends except SRC_OVER as RefuseDiagnostic. Full blend dispatch is a follow-up item.

### API Surface
The `:kanvas` public API is under active development. `@StableApi` declarations are premature and will be re-evaluated once solid-fill rect/rrect/path rendering through the bridge is verified with GPU evidence across reference platforms.

### Explicit Non-Support
- **No dynamic SkSL compilation** — runtime effects use registered Kanvas descriptors with parser-validated WGSL
- **No Ganesh/Graphite port** — WebGPU is the only GPU backend
- **No SkSL compiler/IR/VM** — SkSL is Skia API compatibility vocabulary only
- **No CPU fallback** — unsupported routes emit `RefuseDiagnostic` (bridge `unsupported-skia-bridge-feature` diagnostic wired per M31); no silent CPU rendering
- **No automated rollback detection** — `useLegacyGpuRaster` is a manual flag only

### Codec Support
PNG, JPEG, WebP, GIF, BMP, ICO, WBMP decode; HEIF and AVIF dependency-gated.

### Rollback Procedure
```bash
# Emergency rollback to legacy gpu-raster:
# JVM argument:
-Dkanvas.rollback.legacy-gpu-raster=true
# OR environment variable:
KANVAS_ROLLBACK_LEGACY_GPU_RASTER=true
```
