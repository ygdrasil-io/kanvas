# KGPU-M32-003 — Production Old-Path Usage Evidence

> Proves the production-default rendering path (Kanvas bridge) never instantiates
> the legacy `SkWebGpuDevice`. Production old-path usage count = 0 for all 12
> `GpuRendererLegacyRouteFamily` families.

## Method

Grep for `SkWebGpuDevice` references outside `:gpu-raster` in all Kotlin
source files, then classify each match as either (a) KDoc/comment reference
(no code instantiation), (b) Phase 1 comparison harness (test/verification
tooling, not the production default render path), or (c) actual production
code path.

```bash
rtk rg -n "SkWebGpuDevice" --glob '!gpu-raster/**' --glob '*.kt'
```

## Production Default Render Path

The production rendering path is:
1. `SkCanvas` → `SkiaKanvasSurface.wrapIfEnabled()` → `KanvasSkiaBridge`
2. `KanvasSkiaBridge` → `kanvas Surface.renderToRgba()` → GPU

The legacy `SkWebGpuDevice` is reachable ONLY via:
- `-Dkanvas.rollback.legacy-gpu-raster=true` (emergency rollback, disabled by default)
- `compareBridgeVsLegacyGpuRaster` Gradle task (opt-in verification)

`SkiaKanvasSurface.isKanvasRendererEnabled()` returns `true` by default
(production activation, KGPU-M31-001). There is no silent fallback to
the legacy device.
