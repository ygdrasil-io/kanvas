## Changes Made

### 1. RollbackConfig (`useLegacyGpuRaster` flag)
- **File:** `kanvas-skia-bridge/src/main/kotlin/org/skia/kanvas/RollbackConfig.kt` (NEW)
- `RollbackConfig.useLegacyGpuRaster` reads `kanvas.rollback.legacy-gpu-raster` system property (default `false`)
- Also checks `KANVAS_ROLLBACK_LEGACY_GPU_RASTER` environment variable
- Documented as emergency rollback mechanism

### 2. Default renderer flipped to Kanvas-native
- **File:** `kanvas-skia-bridge/src/main/kotlin/org/skia/kanvas/SkiaKanvasSurface.kt`
- `isKanvasRendererEnabled()` now returns `true` by default (checking `!RollbackConfig.useLegacyGpuRaster`)
- Previously required opt-in via `kanvas.renderer=native`
- `wrapIfEnabled()` now returns non-null by default

### 3. Activation diagnostic
- Added `emitRouteMigratedDiagnostic()` — emits `route-migrated-to-kanvas` diagnostic code once on first bridge activation
- Fires when `wrapIfEnabled()` successfully creates a bridge surface

### 4. gpu-raster deprecation messages updated
- **File:** `gpu-raster/src/main/kotlin/org/skia/gpu/webgpu/SkWebGpuDevice.kt`
- KDoc, `@Deprecated` annotation, and init block all updated to reference `kanvas.rollback.legacy-gpu-raster` instead of the old `kanvas.renderer=native`

### 5. Tests
- `isKanvasRendererEnabled returns true by default` — confirms default is enabled
- `isKanvasRendererEnabled returns false when rollback flag set` — confirms flag works
- `wrapIfEnabled returns non-null by default` — confirms bridge active
- `wrapIfEnabled returns null when useLegacyGpuRaster is set` — confirms rollback works
