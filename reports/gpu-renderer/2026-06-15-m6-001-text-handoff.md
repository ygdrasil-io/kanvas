# GPU Renderer M6-001 Text Handoff

Date: 2026-06-15

## Scope

| Ticket | Final status | Scope |
|---|---|---|
| KGPU-M6-001 | `done` | Accept typed pure Kotlin text artifact references into the renderer `DrawTextRun` payload surface and keep text route execution refused until later M6 gates land. |

## Evidence

- `:gpu-renderer` now depends on `:font:gpu-api` for the renderer-neutral text
  handoff value objects.
- `GPUTextArtifactReference.toRendererTextArtifactRef()` maps font-owned
  artifact type, ID, content fingerprint, generation, and optional route hint
  into `GPUTextArtifactRef`.
- `GPURecorder` still refuses `NormalizedDrawCommand.DrawTextRun` with
  `unsupported.text.draw_run_route_unavailable` and now adds stable terminal
  diagnostics when dumpable payload fields leak Skia-like tokens or a
  CPU-rendered text texture token.
- Tests cover the font-to-renderer artifact reference mapping, the no-`Sk*`
  field-type guard, and recorder diagnostics for text payload leakage.
- Independent review `019ec8d9-9b04-71b0-8d7d-83ce1ffdd94e` verdict:
  `ACCEPT`; no route promotion, adapter/WebGPU claim, forbidden dependency, or
  CPU-rendered fallback was found.

- No A8/SDF/outline/color/bitmap/SVG glyph rendering support is claimed.

  retirement is claimed.
- No CPU-rendered full text texture fallback is allowed.
- KGPU-M6-002 remains blocked on KGPU-M6-003 plus accepted A8 atlas,
