## Changes Made

The `useLegacyGpuRaster` rollback flag was implemented in M30-002. M31-002 formalizes it.

### Rolling flag
- **File:** `kanvas-skia-bridge/src/main/kotlin/org/skia/kanvas/RollbackConfig.kt`
- System property: `-Dkanvas.rollback.legacy-gpu-raster=true`
- Environment variable: `KANVAS_ROLLBACK_LEGACY_GPU_RASTER=true`
- Default: `false` (Kanvas-native pipeline active)
- Integration test: `wrapIfEnabled returns null when useLegacyGpuRaster is set`
- Env var test: `RollbackConfig useLegacyGpuRaster environment variable`

### Rollback procedure
1. Set `-Dkanvas.rollback.legacy-gpu-raster=true` as JVM argument
2. Or set `KANVAS_ROLLBACK_LEGACY_GPU_RASTER=true` as environment variable
3. When active, emits `legacy-gpu-raster-rollback-active` warning diagnostic
4. Legacy `gpu-raster` path (`SkWebGpuDevice`) receives no new features or fixes
