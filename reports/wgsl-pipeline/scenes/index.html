WGSL scene dashboard

Static, data-backed evidence for Kanvas CPU and WebGPU pipeline routes. This dashboard is intentionally separate from the MVP demo: it loads a machine-readable scene registry, links raw artifacts, and keeps expected unsupported diagnostics visible instead of hiding them.

Reference panels

Skia/upstream
The preferred reference. It represents the expected visual output from an upstream-like fixture or imported Skia evidence.
CPU oracle
Used when no upstream image is available. It is a controlled CPU render or oracle path used to compare the GPU lane.
Diff panels
Diff images compare CPU and GPU outputs against the reference when the artifact exists. Missing GPU diffs stay visible as tracked gaps.
Not parity
A reference panel proves the selected scene contract only. It does not imply full Skia compatibility for neighboring APIs.

Route diagnostics

CPU route
The reference execution path used by the CPU lane. It proves which coverage, shader, image, or blend path produced the oracle artifact.
GPU route
The selected WebGPU pipeline route or coverage strategy. A passing scene should expose the same semantic route family as the CPU evidence.
Fallback
none means the GPU lane rendered normally. Any other value is a stable diagnostic and must stay visible, especially for tracked gaps and expected unsupported rows.
Tracked gap
The route is documented, but at least one required visual capture is missing. This is not counted as a pass.
Expected unsupported
The scene is intentionally refused by the current GPU policy. It remains in the dashboard so breadth limits are visible instead of hidden.
